Installation

Five steps to a better prompt

Run sudo bash setup.sh to automate everything below — or follow the steps manually.

1

Uninstall Oh My Zsh

Removes the Oh My Zsh framework and its folder if present. Safe to skip if it was never installed.

bash 
# Run the built-in uninstaller if it exists
sh ~/.oh-my-zsh/tools/uninstall.sh
rm -rf ~/.oh-my-zsh
2

Set default shell to zsh

Makes zsh your login shell so the Starship initialisation in .zshrc is picked up on every new terminal.

bash 
chsh -s /usr/bin/zsh
3

Install Starship

Installs the Starship binary from the Debian/Ubuntu package repository. Already installed? This is a no-op.

bash 
sudo apt install -y starship
4

Create ~/.zshrc

Sets up aliases, launches fastfetch on every new shell, and initialises Starship. The old .zshrc is backed up to .zshrc.bak.

zsh 
# CPU temperature (Raspberry Pi)
alias temp="echo \$((\`cat /sys/class/thermal/thermal_zone0/temp|cut -c1-2\`)).\$((\`cat /sys/class/thermal/thermal_zone0/temp|cut -c3-5\`))"

# bat (better cat) alias
alias bat="batcat"

# Show system info on shell start
fastfetch

# Initialise Starship prompt
eval "$(starship init zsh)"
ℹ️ The eval "$(starship init zsh)" line must be the last entry in your .zshrc to ensure it wraps the final prompt correctly.
5

Install starship.toml

Copies the configuration file from this repository to ~/.config/starship.toml where Starship reads it automatically.

bash 
mkdir -p ~/.config
cp starship.toml ~/.config/starship.toml
bash — run everything at once 
sudo bash setup.sh
⚠️ After the script finishes, open a new terminal (or run exec zsh) to activate the new prompt.

Configuration

starship.toml

Drop this file at ~/.config/starship.toml. Every module is opt-in — comment out anything you don't need.

toml 
# Prompt layout: all modules → custom → newline → character
format = "$all$custom$line_break$character"

# ── Local IP ─────────────────────────────────────
[localip]
ssh_only = false
format   = '@[$localipv4](bold red) '
disabled = false

# ── Directory ────────────────────────────────────
[directory]
truncation_length = 8
truncation_symbol = '…/'

# ── Memory usage ─────────────────────────────────
[memory_usage]
disabled  = false
threshold = -1
symbol    = ' '
style     = 'bold dimmed green'

# ── Git status ───────────────────────────────────
[git_status]
conflicted = '🏳'
ahead      = '🏎💨'
behind     = '😰'
diverged   = '😵'
up_to_date = '✓'
untracked  = '🤷'
stashed    = '📦'
modified   = '📝'
staged     = '[++\($count\)](green)'
renamed    = '👅'
deleted    = '🗑'

# ── Custom: uptime ───────────────────────────────
[custom.uptime]
command = "uptime -p | sed 's/up //' | sed 's/ days\\?,/d/' | sed 's/ hours\\?,/h/' | sed 's/ minutes\\?/m/'"
when    = true
symbol  = "up "
style   = "bold yellow"
format  = "[$symbol$output]($style) "

# ── Custom: CPU temperature (Raspberry Pi — vcgencmd) ───
[custom.temp]
command = "vcgencmd measure_temp | sed \"s/temp=//;s/'C//\""
when    = "command -v vcgencmd > /dev/null 2>&1"
symbol  = "T:"
style   = "bold red"
format  = "[$symbol${output}°C]($style) "

# ── Custom: CPU temperature (RISC-V 64 / generic Linux — sysfs) ─
# Auto-hides on Raspberry Pi (where vcgencmd takes precedence)
# To find the right zone: for z in /sys/class/thermal/thermal_zone*/temp; do echo "$z: $(cat $z)"; done
[custom.temp_riscv]
command = "awk '{printf \"%.0f\", $1/1000}' /sys/class/thermal/thermal_zone0/temp"
when    = "test -f /sys/class/thermal/thermal_zone0/temp && ! command -v vcgencmd > /dev/null 2>&1"
symbol  = "T:"
style   = "bold red"
format  = "[$symbol${output}°C]($style) "

Prompt Modules

What's in your prompt

Each module is independently configurable. Disable any you don't need.

🌐
Local IP
Shows your machine's IPv4 address — always visible, not just in SSH sessions.
📁
Directory
Truncates deep paths to 8 levels with an …/ prefix so you always know where you are.
🧠
Memory Usage
Displays RAM usage in bold green. Threshold set to -1 so it's always shown.
🔀
Git Status
Rich emoji indicators for every git state — ahead, behind, stashed, modified, staged, renamed, and more.
⏱
Uptime
Custom module showing system uptime in a compact format like up 2d 4h 30m.
🌡
CPU Temperature
Uses vcgencmd on Raspberry Pi or the Linux sysfs interface on RISC-V 64 and other boards. Each module auto-hides when its method is unavailable.
